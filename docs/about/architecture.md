# Architecture

The Sovereign Codex is built as a layered documentation system.

Each layer has a distinct responsibility, allowing the project to remain organised as it grows.

---

## Foundation

MkDocs provides the documentation engine.

Material for MkDocs provides the user interface.

Git provides version control.

GitHub Pages provides publishing.

---

## Information Architecture

Documentation is organised into distinct knowledge domains.

Each domain has a clearly defined purpose and ownership.

This reduces ambiguity and improves long-term maintainability.

---

## Design System

Visual consistency is maintained through a semantic design system.

Rather than styling individual components directly, reusable design tokens define colours, spacing, typography, borders, shadows, and interaction.

---

## Governance

Content flows through governance and quality review before becoming trusted knowledge.

This ensures published information remains accurate and maintainable.

---

## Philosophy

The architecture prioritises clarity over complexity, systems over shortcuts, and long-term maintainability over rapid expansion.
