# Naming Standard

The Naming Standard establishes the conventions used when naming documents, folders, headings, and other project assets within Sovereign Codex.

Its purpose is to promote consistency, improve navigation, and make information easier to locate and maintain over time.

Names should prioritise clarity and meaning over creativity or brevity.

---

## Core Principles

Names should be:

- Clear
- Descriptive
- Consistent
- Predictable
- Maintainable

---

## Documents

Document titles should clearly describe their purpose.

Readers should be able to understand the subject of a document without opening it.

Avoid ambiguous or overly broad titles.

---

## Folders

Folders should represent a single discipline or logical grouping of related documentation.

Folder names should remain stable and should only change when there is a clear architectural benefit.

---

## Headings

Headings should accurately describe the content that follows.

Similar topics should use consistent heading names throughout the project wherever practical.

---

## Terminology

Terminology should remain consistent across the project.

Where multiple names exist for the same concept, a single preferred term should be adopted and used consistently.

---

## File Names

File names should be:

- Lowercase
- Descriptive
- Hyphen-separated
- Easy to read

Avoid spaces, special characters, and unnecessary abbreviations.

---

## Continuous Improvement

Naming conventions should evolve only when they improve clarity, consistency, or long-term maintainability.

Changes should be deliberate and applied consistently throughout the project.
