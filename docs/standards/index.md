# Standards

The Standards section defines the conventions used when creating and maintaining documentation within Sovereign Codex.

These standards promote consistency across the project by establishing common approaches to writing, formatting, organising, and maintaining documentation.

Unlike Governance, which defines the principles and processes of the project, the Standards describe how documentation should be produced in practice.

---

## Scope

The Standards establish guidance for:

- Documentation structure
- Naming conventions
- Writing style
- Markdown formatting
- Evidence and citations
- Page templates
- Versioning practices

Together, these standards help ensure that documentation remains clear, consistent, and maintainable as the project grows.

---

## Standards

- **Project Standards** — General documentation expectations.
- **Naming Conventions** — Naming rules for pages, folders, and headings.
- **Markdown Style** — Formatting and writing conventions.
- **Evidence & Citations** — Referencing and supporting information.
- **Templates** — Standard page structures.
- **Versioning** — Documentation version management.
