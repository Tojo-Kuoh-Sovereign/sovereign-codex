# Metadata Standard

The Metadata Standard defines the information that accompanies documentation published within Sovereign Codex.

Its purpose is to improve organisation, discoverability, maintainability, and lifecycle management by providing consistent information about each document.

Metadata should support both contributors and readers without distracting from the primary content.

---

## Core Principles

Metadata should be:

- Consistent
- Useful
- Accurate
- Maintainable
- Relevant

---

## Purpose

Metadata provides important context that may not naturally form part of the document itself.

Examples include publication status, version information, review dates, and document ownership.

Consistent metadata improves navigation, maintenance, and long-term governance.

---

## Recommended Metadata

Where appropriate, documents should record:

- Title
- Description
- Version
- Status
- Author
- Last Reviewed
- Category
- Tags
- Confidence Level (where applicable)

Not every document requires every metadata field.

Metadata should remain appropriate to the purpose of the document.

---

## Document Status

Documentation may use publication status to communicate its current stage within the documentation lifecycle.

Common examples include:

- Draft
- Review
- Published
- Archived

Status should accurately reflect the maturity of the document.

---

## Version Information

Version information should follow the project's Versioning Standard.

Where version numbers are used, they should accurately represent meaningful changes to the document.

---

## Review Information

Where appropriate, documentation should record when it was last reviewed.

Review information supports ongoing maintenance and helps identify documentation requiring future validation.

---

## Continuous Improvement

Metadata requirements should evolve alongside the project.

Changes should improve organisation, discoverability, and long-term maintainability without creating unnecessary administrative overhead.
