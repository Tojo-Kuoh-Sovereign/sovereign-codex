# Purpose

The Navigation Standard defines how users discover, navigate, and understand knowledge throughout the Sovereign Codex.

Navigation is designed to minimise cognitive load by exposing meaningful knowledge domains while hiding repository implementation details.

Navigation supports discovery, learning progression and contextual understanding.

---

# Objectives

This standard implements the following Design Principles:

- Support rapid discovery of knowledge.
- Present information in a logical hierarchy.
- Reinforce relationships between knowledge domains.
- Reduce unnecessary cognitive effort.
- Remain scalable as the repository grows.

---

# Standard

Only expose information necessary for the current level of exploration. 

Equivalent knowledge should appear in equivalent locations. 

Users should be able to reasonably predict where information will be located.

Navigation should favour recognisable terminology over requiring users to remember internal structures

---

# Navigation Principles

Navigation within the Sovereign Codex should be guided by the following principles:

- Progressive Disclosure
- Consistency
- Predictability
- Recognition over Recall
- Domain-Oriented Navigation

# Navigation Hierarchy

Navigation is organised around knowledge domains rather than repository folders. 

## Knowledge Domains

Represent the primary areas of responsibility within the Sovereign Codex.
Examples include Governance, Research, Standards, and Design.

## Knowledge Categories

Group related concepts within a knowledge domain.
Categories provide structure while limiting navigation depth.

## Knowledge Articles

Represent individual knowledge artefacts.
Articles communicate a single concept, standard, guide, or reference.

---

# Navigation Labels

Navigation labels should:

- Be concise.
- Use Title Case.
- Describe knowledge domains.
- Avoid implementation terminology.
- Remain stable over time.
- Favour recognition over abbreviation.
- Remain meaningful when viewed independently.

---

# Relationship to Repository Naming

Navigation labels are governed independently from repository naming conventions.

Repository naming exists to support contributor workflows.

Navigation labels exist to support reader comprehension.

The Repository Naming Standard defines internal folder conventions.

This Navigation Standard defines how those structures are presented to users.

---

# Governance

Document structures should evolve only where changes improve discoverability, clarity, consistency, or user experience.

Changes should remain aligned with the Information Architecture and Sovereign Governance framework.
