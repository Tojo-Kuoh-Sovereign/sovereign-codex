# Typography

## Purpose

Typography establishes the reading experience of Sovereign Codex.

As a documentation platform, readability is the highest priority.

Typography should support learning through consistency, hierarchy and clarity.

---

# Philosophy

Typography should be:

- Clear
- Comfortable
- Predictable
- Consistent
- Professional

The platform should encourage long-form reading without fatigue.

---

# Hierarchy

Every page should maintain a consistent heading structure.

# Heading 1

Document title.

Used once per page.

---

## Heading 2

Primary sections.

---

### Heading 3

Subsections.

---

#### Heading 4

Supporting information.

Additional heading levels should be avoided whenever possible.

---

# Paragraphs

Paragraphs should remain concise.

Large walls of text should be broken into logical sections.

Lists should be used where appropriate to improve readability.

---

# Code

Code blocks should remain visually distinct from surrounding content.

Syntax highlighting should support readability without becoming distracting.

---

# Tables

Tables should be used for:

- Comparisons
- Reference information
- Statistics
- Technical specifications

Tables should not replace explanatory text.

---

# Quotes

Quotes should be reserved for:

- Mission statements
- Principles
- Official references

They should emphasise ideas rather than decoration.

---

# Emphasis

Bold should communicate importance.

Italic should communicate emphasis.

Excessive formatting should be avoided.

---

# Reading Experience

Typography should support:

- Long-form documentation
- Technical reference
- Quick scanning
- Mobile readability

---

# Success Criteria

Typography should:

- Improve comprehension.
- Reduce reading fatigue.
- Maintain consistency.
- Clearly communicate hierarchy.
