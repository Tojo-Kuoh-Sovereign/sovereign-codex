# Purpose

The Metadata Standard defines the metadata used throughout the Sovereign Documentation Framework to improve organisation, discoverability, maintainability, and lifecycle management.

Consistent metadata provides readers and contributors with important contextual information while supporting long-term governance of the repository.

---

# Relationships

This standard implements the following Governance principles:

- Preserve Orientation
- Reduce Unnecessary Cognitive Effort
- Build Confidence Through Clarity
- Continuous Improvement

---

# Standard

Metadata should communicate the administrative context of a document without distracting from its primary purpose.

Metadata should remain consistent throughout the repository while only including information that provides meaningful value to contributors or readers.

Not every document requires every metadata field.

Metadata should always remain appropriate to the responsibility of the document.

---

# Standard Metadata

Governance, Research, and Standards documents should normally include:

- Status
- Document Owner
- Framework
- Category
- Version

Example:

```markdown
> **Status:** Draft
>
> **Document Owner:** Sovereign Project Office
>
> **Framework:** Sovereign Documentation Framework (SDF)
>
> **Category:** Standards
>
> **Version:** 1.0
```

---

# Optional Metadata

Additional metadata may be included where appropriate.

Examples include:

- Review Date
- Confidence Level
- Dependencies
- Related Documents
- Tags

Optional metadata should only be included where it improves the usability or governance of the document.

---

# Status Values

Status communicates the current maturity of a document.

Typical values include:

- Draft
- Review
- Approved
- Archived

Status should accurately represent the document's current stage within the documentation lifecycle.

---

# Version Information

Versioning should follow the Sovereign Versioning Standard.

Version numbers should represent meaningful changes to the document rather than routine editorial updates.

---

# Governance

Metadata should support governance without creating unnecessary administrative overhead.

Additional metadata fields should only be introduced where they improve organisation, discoverability, or long-term maintainability.
