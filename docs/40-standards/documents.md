# Purpose

The Document Standard defines the structure and organisation of documentation throughout the Sovereign Documentation Framework.

Its purpose is to provide a consistent reading experience while ensuring every document clearly communicates its responsibility within the framework.

---

# Relationships

This standard implements the following Governance principles:

- Progressive Engagement
- Preserve Orientation
- Reduce Unnecessary Cognitive Effort
- Build Confidence Through Clarity

---

# Standard

Every document should communicate a single, clearly defined responsibility.

Documentation should remain consistent in structure while allowing sufficient flexibility to accommodate different document types throughout the Sovereign Documentation Framework.

Consistency should improve readability, maintainability, and contributor orientation without unnecessarily restricting documentation.

---

# Canonical Structure

Where appropriate, documents should generally follow the following structure:

1. Metadata
2. Purpose
3. Relationships
4. Primary Content
5. Governance (where applicable)
6. Version History

Not every document requires every section.

The structure should always reflect the responsibility of the document while remaining consistent with the Sovereign Documentation Framework.

---

# Canonical Documents

Different areas of the Sovereign Documentation Framework communicate different responsibilities.

| Framework Area | Canonical Document |
|----------------|-------------------|
| Governance | Statement |
| Research | Statement |
| Standards | Standard |
| Design | Specification |
| Quality | Review |
| Knowledge | Article |

Supporting documents may exist where additional implementation guidance, examples, templates, or reference material are required.

---

# Document Responsibilities

Documents should communicate a single responsibility wherever practical.

Examples include:

- Statement
- Standard
- Specification
- Review
- Article
- Guide
- Template

Document names should accurately reflect their responsibility within the framework.

---

# Governance

Document structures should evolve only where changes improve clarity, consistency, maintainability, or contributor experience.

Structural changes should remain aligned with the Sovereign Governance framework and avoid unnecessary complexity.
