# Purpose

The Metadata Standard defines the metadata used throughout the Sovereign Documentation Framework to improve organisation, discoverability, maintainability, and document lifecycle management.

Consistent metadata provides readers and contributors with meaningful contextual information while supporting the long-term governance of the framework.

---

# Relationships

This standard implements:

- Preserve Orientation
- Reduce Unnecessary Cognitive Effort
- Build Confidence Through Clarity

This standard supports:

- Documentation Standards
- Versioning
- Repository Organisation
- Governance Documents
- Research Documents

---

# Standard

Metadata should communicate the administrative context of a document without distracting from its primary purpose.

Metadata should remain consistent throughout the framework while only including information that provides meaningful value to readers or contributors.

Not every document requires every metadata field.

Metadata should always remain appropriate to the responsibility of the document.

---

# Canonical Metadata

Governance, Research, and Standards documents should normally include:

- Status
- Document Owner
- Framework
- Category
- Version

Example:

```markdown
> **Status:** Draft
>
> **Document Owner:** Sovereign Project Office
>
> **Framework:** Sovereign Documentation Framework (SDF)
>
> **Category:** Standards
>
> **Version:** 1.0
```

---

# Optional Metadata

Additional metadata may be included where appropriate.

Examples include:

- Review Date
- Confidence Level
- Dependencies
- Related Documents
- Tags

Optional metadata should only be included where it improves organisation, governance, discoverability, or maintainability.

---

# Status Values

Status communicates the current maturity of a document.

Typical values include:

- Draft
- Review
- Approved
- Archived

Status should accurately represent the document's current stage within the documentation lifecycle.

---

# Version Information

Versioning should follow the Sovereign Versioning Standard.

Version numbers should represent meaningful changes to the document rather than routine editorial updates.

---

# Governance

The Metadata Standard should evolve only where changes improve organisation, discoverability, maintainability, or contributor experience.

Additional metadata fields should only be introduced where they communicate genuinely useful contextual information without creating unnecessary administrative overhead.
