# Purpose

The Navigation Standard defines how readers discover, navigate, and understand knowledge throughout the Sovereign Documentation Framework.

Its purpose is to minimise cognitive effort by presenting meaningful knowledge domains while hiding repository implementation details.

Navigation should support discovery, learning progression, and contextual understanding.

---

# Standard

Navigation should expose only the information necessary for the reader's current level of exploration.

Equivalent knowledge should appear in equivalent locations.

Readers should be able to reasonably predict where information will be located.

Navigation should favour recognisable terminology over internal implementation details.

---

# Navigation Principles

Navigation throughout the Sovereign Documentation Framework should be guided by the following principles.

## Progressive Disclosure

Present information gradually.

Readers should encounter increasing levels of detail only as they move deeper into the framework.

---

## Consistency

Equivalent concepts should always appear in equivalent locations.

Consistent navigation reduces cognitive effort and improves confidence.

---

## Predictability

Readers should be able to anticipate where information is likely to be found.

Unexpected organisation increases navigation effort.

---

## Recognition over Recall

Navigation should minimise the need to remember repository structures, internal terminology, or implementation details.

Readers should recognise knowledge rather than recall where it is stored.

---

## Domain-Oriented Navigation

Navigation should reflect knowledge domains rather than technical implementation.

Knowledge should remain the organising principle throughout the framework.

---

# Navigation Hierarchy

Navigation is organised around knowledge domains rather than repository folders.

## Knowledge Domains

Represent the primary areas of responsibility within the Sovereign Documentation Framework.

Examples include:

- Governance
- Research
- Standards
- Architecture
- Knowledge

---

## Categories

Group related concepts within a knowledge domain.

Categories provide structure while avoiding unnecessary navigation depth.

---

## Documents

Represent individual knowledge artefacts.

Each document should communicate a single responsibility within its domain.

---

# Navigation Labels

Navigation labels should:

- Be concise.
- Use Title Case.
- Describe knowledge rather than implementation.
- Avoid internal terminology.
- Remain meaningful when viewed independently.
- Favour recognition over abbreviation.
- Remain stable over time.

---

# Relationship to Repository Naming

Navigation labels are governed independently from repository naming conventions.

Repository naming supports contributor workflows.

Navigation supports reader understanding.

The Repository Naming Standard defines how documentation is organised internally.

The Navigation Standard defines how that organisation is presented to readers.

---

# Governance

The Navigation Standard should evolve only where changes improve discoverability, clarity, consistency, accessibility, or reader experience.

Changes should remain aligned with the Information Architecture and Sovereign Governance Framework.
