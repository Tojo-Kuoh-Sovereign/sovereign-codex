# Purpose

The Iconography Specification defines the principles governing the selection, design, and use of icons throughout the Sovereign Documentation Framework.

Its purpose is to improve recognition, navigation, orientation, and comprehension while remaining consistent with the Sovereign Design Language.

Icons should reinforce understanding without distracting from the knowledge they support.

# Specification

Icons should reinforce meaning rather than replace it.

They should improve recognition, navigation, and visual scanning while remaining secondary to written communication.

Every icon should communicate a single, well-understood purpose and contribute positively to the reader's understanding.

---

# Iconography Philosophy

Icons should be:

- Purposeful
- Recognisable
- Consistent
- Accessible
- Minimal

Icons communicate supporting meaning.

They should never become the primary method of communicating important information.

---

# Roles of Icons

Icons may be used to:

- Support navigation
- Identify actions
- Reinforce information hierarchy
- Improve visual scanning
- Represent familiar concepts
- Communicate Knowledge States
- Support interactive elements

Icons should never be introduced solely for decoration.

---

# Consistency

Icons should maintain a consistent visual language throughout the platform.

Consistency should include:

- Style
- Weight
- Size
- Alignment
- Spacing
- Placement

Readers should quickly recognise familiar iconography regardless of where it appears.

---

# Labels

Icons should complement text rather than replace it.

Where meaning may be ambiguous, icons should always be accompanied by descriptive labels or surrounding context.

Readers should never be expected to interpret unfamiliar symbols without explanation.

---

# Accessibility

Icons should never be the sole method of communicating important information.

Meaning should always be reinforced through:

- Text
- Labels
- Structure
- Additional visual cues

Accessible iconography improves usability for every reader.

---

# Restraint

Icons should be introduced only where they improve understanding, navigation, or recognition.

Excessive icon usage creates visual noise, reduces hierarchy, and weakens the effectiveness of icons that genuinely communicate meaning.

Every icon should earn its place.

---

# Governance

The Iconography Specification should evolve through implementation experience, usability observations, accessibility improvements, and continuous refinement.

Changes should strengthen consistency, clarity, and recognition while remaining aligned with the Sovereign Design Language and Knowledge Presentation Principles.
