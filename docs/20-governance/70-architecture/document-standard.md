# Purpose

The Document Standard defines the structure, organisation, and responsibilities of documentation throughout the Sovereign Documentation Framework.

Its purpose is to provide a consistent reading experience while ensuring every document clearly communicates its role within the framework.

---

# Relationships

This standard implements the following Knowledge Presentation Principles:

- Progressive Engagement
- Preserve Orientation
- Reduce Unnecessary Cognitive Effort
- Build Confidence Through Clarity

This standard supports:

- Governance Statements
- Research Documents
- Presentation Standards
- Design Specifications
- Quality Reviews
- Knowledge Articles

---

# Standard

Every document should communicate a single, clearly defined responsibility.

Documentation should remain consistent in structure while allowing sufficient flexibility to support different document types throughout the Sovereign Documentation Framework.

Consistency should improve readability, maintainability, navigation, and contributor experience without unnecessarily restricting documentation.

---

# Canonical Structure

Where appropriate, documents should normally follow the following structure:

1. Metadata
2. Purpose
3. Relationships
4. Primary Content
5. Governance (where applicable)
6. Version History

Not every document requires every section.

The structure should always reflect the responsibility of the document while remaining consistent with the Sovereign Documentation Framework.

---

# Document Responsibilities

Readers should be able to understand both the purpose and responsibility of a document from its title alone.

Common document responsibilities include:

- Statement
- Standard
- Specification
- Review
- Article
- Guide
- Template
- Reference

Document titles should accurately reflect their responsibility within the framework.

Readers should be able to understand the purpose of a document from its title alone.

---

# Consistency

Documentation should remain internally consistent in:

- Structure
- Metadata
- Heading hierarchy
- Terminology
- Relationships
- Navigation

Consistency improves readability, contributor experience, and long-term maintainability.

---

# Governance

Document structures should evolve only where changes improve clarity, consistency, maintainability, accessibility, or contributor experience.

Structural changes should remain aligned with the Sovereign Documentation Framework and avoid introducing unnecessary complexity.
