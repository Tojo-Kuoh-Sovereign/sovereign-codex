# Purpose

The Typography Standard defines how written information is presented throughout the Sovereign Documentation Framework.

Its purpose is to establish a consistent reading experience that supports learning, technical reference, long-form documentation, and knowledge discovery.

Typography should improve comprehension through hierarchy, rhythm, and readability rather than visual expression.

# Standard

Typography exists to organise information rather than decorate it.

Clear hierarchy allows readers to understand relationships between ideas, quickly locate relevant information, and comfortably navigate complex documentation.

Good typography becomes almost invisible, allowing knowledge to remain the primary focus.

---

# Typography Philosophy

Typography should communicate:

- Clarity
- Structure
- Consistency
- Confidence
- Readability

The reading experience should feel calm, deliberate, and effortless.

---

# Information Hierarchy

Typography establishes the structure of every document.

Each heading level should communicate a clear relationship to the content beneath it.

Readers should be able to understand the organisation of a page by scanning its headings alone.

Hierarchy should remain consistent throughout the platform.

---

# Reading Experience

Documentation should encourage comfortable long-form reading.

Content should be organised into logical sections using concise paragraphs, descriptive headings, lists, tables, and callouts where appropriate.

Whitespace should be treated as part of the reading experience rather than unused space.

---

# Emphasis

Visual emphasis should support understanding rather than decoration.

- **Bold** should identify importance.
- *Italic* should provide emphasis where appropriate.
- Excessive formatting should be avoided.

When everything is emphasised, nothing is emphasised.

---

# Technical Content

Technical information should remain visually distinct while integrating naturally with surrounding documentation.

Examples include:

- Code blocks
- Commands
- File paths
- Keyboard shortcuts
- Configuration snippets

Formatting should improve recognition without interrupting reading flow.

---

# Tables

Tables should be used when information benefits from structured comparison or reference.

They should complement explanatory text rather than replace it.

Readers should be able to identify relationships and differences quickly.

---

# Accessibility

Typography should support readers across a wide range of devices and reading conditions.

Hierarchy, spacing, and contrast should improve comprehension without relying on colour or decorative styling.

The objective is to reduce cognitive effort while preserving technical accuracy.

---

# Governance

The Typography Standard should evolve only where changes improve readability, accessibility, consistency, or long-term maintainability.

Changes should remain aligned with the Design System, Knowledge Presentation Principles, and Identity of the Sovereign Documentation Framework.
