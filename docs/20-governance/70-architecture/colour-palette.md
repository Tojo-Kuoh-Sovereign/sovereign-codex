# Purpose

The Colour Palette defines the visual foundation of the Sovereign Documentation Framework.

Its purpose is to establish a consistent, meaningful, and accessible system of colours that communicates hierarchy, reinforces identity, and supports long-form knowledge presentation.

Rather than selecting colours for decoration, the palette is derived from materials that represent the values, permanence, and craftsmanship of the Sovereign ecosystem.

# Statement

Colour should communicate meaning before aesthetics.

Every colour within the Sovereign Documentation Framework should reinforce clarity, hierarchy, trust, and usability while contributing to a consistent visual identity.

The palette should remain restrained, timeless, and purposeful rather than following short-term design trends.

---

# Colour Philosophy

The Sovereign Documentation Framework begins with materials rather than colours.

Each material represents an enduring characteristic of the platform and is translated into a digital colour palette.

The interface becomes a contemporary interpretation of enduring materials rather than an arbitrary collection of colours.

---

# Material Palette

| Material | Represents | Typical Usage |
|----------|------------|---------------|
| **Obsidian** | Foundation, permanence | Primary backgrounds |
| **Forged Steel** | Structure, engineering | Borders, navigation, secondary surfaces |
| **Bronze** | Stewardship, craftsmanship | Headings, accents, emphasis |
| **Sapphire** | Knowledge, trust | Interactive elements, links, active navigation |
| **Paper** | Human knowledge | Primary text and readable content |

Each material contributes to the overall character of the platform while serving a clearly defined functional purpose.

---

# Semantic Colour System

Implementation should reference colours according to their semantic responsibility rather than their visual appearance.

Examples include:

- Surface
- Border
- Text
- Heading
- Interactive
- Success
- Warning
- Critical

Using semantic colour roles improves maintainability while allowing the visual identity to evolve without affecting implementation.

---

# Hierarchy

Colour should establish visual hierarchy before attracting attention.

Primary information should remain the visual focus.

Accent colours should guide the reader rather than dominate the interface.

Restraint strengthens clarity.

---

# Accessibility

The colour palette should support comfortable long-form reading.

Contrast should remain sufficient for readability while avoiding unnecessary visual fatigue.

Colour should never be the sole method of communicating information.

Accessibility should remain a fundamental consideration throughout the evolution of the palette.

---

# Governance

The Colour Palette should evolve only where changes strengthen clarity, accessibility, consistency, or the long-term identity of the Sovereign Documentation Framework.

New colours should be introduced only where they communicate genuinely new semantic meaning rather than additional visual decoration.

Changes should remain consistent with the Design System and preserve a recognisable visual language throughout the Sovereign ecosystem.
