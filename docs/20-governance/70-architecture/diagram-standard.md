# Purpose

The Diagram Standard defines the principles and conventions used when creating visual diagrams throughout the Sovereign Documentation Framework.

Its purpose is to ensure diagrams communicate information clearly, consistently, accurately, and accessibly while supporting the broader Knowledge Presentation Principles.

Diagrams should simplify complex information rather than introduce unnecessary visual complexity.

# Standard

Diagrams exist to improve understanding where visual communication provides greater clarity than text alone.

They should complement written documentation rather than replace it.

Every diagram should communicate a single primary idea and support the reader's current objective.

---

# Diagram Types

Where appropriate, documentation may include:

- Flow Diagrams
- Process Diagrams
- Architecture Diagrams
- Decision Trees
- Relationship Diagrams
- Timelines
- Concept Maps
- Knowledge Hierarchies

Each diagram should have a clearly defined purpose.

---

# Design Principles

Diagrams should:

- Use consistent terminology.
- Follow a logical reading order.
- Minimise unnecessary visual complexity.
- Prioritise readability over decoration.
- Present relationships clearly.
- Remain consistent with the Sovereign Design System.

Visual design should reinforce understanding rather than attract attention to itself.

---

# Accessibility

Diagrams should remain understandable when viewed independently.

Supporting text should provide sufficient context for readers who may be unable to interpret the visual information alone.

Where practical:

- Use descriptive titles.
- Label important relationships.
- Avoid relying solely on colour.
- Ensure diagrams remain understandable when printed or viewed in monochrome.

---

# Governance

Diagram standards should evolve through research, implementation experience, and continuous improvement.

Changes should improve clarity, consistency, accessibility, and maintainability while remaining aligned with the Knowledge Presentation Principles.
